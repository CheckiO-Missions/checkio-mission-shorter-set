Since set is an unordered collection, you need to create new ordered collection, which can be sorted. The list object is fine. So, first you need sorted() function to create sorted list from your input set. 
data = {1, 5, 2, 4}
order = sorted(data)
print(order) # [1, 2, 4, 5] you got list and it is sorted ascending.
And how to remove amount of minimum and maximum values from the list?
Your list is sorted ascending, so it starts from smallest values and ends with biggest. Now you may use slicing with positive and negative indexes to get partial list. 
order_sl = order[1:-1] # you get new list from previous with
                    # sliced off first and -first(last) values
print(order_sl) # [2, 4]
Ok. I still don't understand, what indexes to use in my solution for slicing.
You have input value 'total'. It shows how many smallest and biggest numbers should be deleted from set. Use it as positive and negative index to slice them off your sorted list. 
order = sorted(data)
order_sl = order[total:-total]
After previous step I have a list, but the solution must return set. What should I do?
Yes. You just need to create set from your list with set(order_sl) and return it. 
order = sorted(data)
order_sl = order[total:-total]

return set(order_sl)
And the final question. My slicing doesn't work if total == 0. How to include this case in solution?
You have to check 'total' value before other commands to avoid doing unnecessary calculations. You may do this with 'if total == 0:', but the pythonic way to do this is just 'if not total:'. Since there is no amount to remove, you return the initial set in this case. And don't forget to declare function itself. 
def remove_min_max(data: set[int], total:int) -> set[int]:
    
    if not total:
        return data

    order = sorted(data)
    order_sl = order[total:-total]

    return set(order_sl)
